# ICM45686 示例工程

本目录提供 ICM45686 在 MSPM0 和 STM32 平台上的参考工程，并按串口输出格式分为两个版本：

- `纯串口输出/`：保留原始示例，通过串口输出可直接阅读的文本数据。
- `JYTech上位机输出/`：通过串口输出 JYTech AHRS Studio 可解析的 56 字节二进制数据帧。

两个目录中的传感器初始化、I2C/SPI 驱动和硬件接线方式保持一致，区别仅在应用层串口输出格式。请勿使用普通串口助手的文本模式查看 JYTech 版本，否则会显示乱码。

## 支持的工程

| 平台 | 开发环境 | 接口 |
| --- | --- | --- |
| MSPM0G3507 | Code Composer Studio | I2C、SPI |
| MSPM0G3507 | Keil MDK | I2C、SPI |
| STM32F103 | Keil MDK | I2C、SPI |
| STM32F407 | Keil MDK/HAL | 由工程配置选择 |
| Arduino | Arduino IDE | I2C |

各子工程的具体接线和导入方法见其内部 `README.md`。

## 纯串口输出

纯串口版本原封不动保留原示例行为，默认使用 `115200 8N1`，周期输出以下物理量：

- 三轴加速度，单位 `mg`
- 三轴角速度，单位 `°/s`
- 温度，单位 `°C`（Arduino 原始示例除外）

该版本适合使用串口助手观察数据、验证传感器接线和移植底层驱动。

## JYTech 上位机输出

JYTech 版本默认使用 `115200 8N1`，不输出启动提示或调试文本，只连续发送固定 56 字节二进制帧。在 JYTech AHRS Studio 启动时请选择 **ICM45686** 产品。

### 帧格式

| 偏移 | 字段 | 类型 | 说明 |
| --- | --- | --- | --- |
| 0~1 | 帧头 | `uint8_t[2]` | 固定 `AA 55` |
| 2 | 数据类型 | `uint8_t` | 固定 `02` |
| 3 | 长度 | `uint8_t` | 固定 `36`（十进制 54） |
| 4~9 | Accel X/Y/Z | `int16_t × 3` | 小端序，`2048 LSB/g` |
| 10~15 | Gyro X/Y/Z | `int16_t × 3` | 小端序，`16.4 LSB/(°/s)` |
| 16~27 | Roll/Pitch/Yaw | `float × 3` | 示例不执行姿态融合，固定填 `0.0f` |
| 28~29 | Temperature | `int16_t` | 小端序，实际温度为数值除以 100，单位 `°C` |
| 30~41 | Raw Accel X/Y/Z | `float × 3` | 小端序，单位 `g` |
| 42~53 | Raw Gyro X/Y/Z | `float × 3` | 小端序，单位 `°/s` |
| 54 | 校验和 | `uint8_t` | 字节 2~53 的累加和低 8 位 |
| 55 | 帧尾 | `uint8_t` | 固定 `5A` |

完整协议及下行命令定义见仓库根目录的 [`communication_protocol.md`](../communication_protocol.md)。

### 输出频率

各工程沿用原示例循环周期：

- MSPM0 CCS：约 20 Hz
- MSPM0 Keil、STM32F103：约 100 Hz
- STM32F407、Arduino：约 50 Hz

这些示例仅负责采集 ICM45686 并向上位机发送加速度、角速度和温度，不包含 G356 固件中的校准、Mahony 姿态融合、Flash 配置或下行控制命令处理。

## 使用注意

1. 根据硬件连接选择正确的 I2C 或 SPI 工程。
2. 确认串口波特率为 115200。
3. JYTech 版本必须以二进制方式发送完整数据帧，不要在同一串口插入 `printf`、`Serial.print` 等文本。
4. 所有多字节字段均按小端序发送，目标 MCU 和上位机需使用 IEEE 754 单精度浮点格式。
5. 修改量程后必须同步调整加速度和角速度换算系数，否则上位机显示值会不正确。
